أنشئ نظام سلسلة العمل (Workflow) للقضايا والاستشارات:

=== 1. Schema في shared/schema.ts ===

- enum ClientRole: plaintiff (مدعي), defendant (مدعى عليه)

- enum CasePriority: urgent (عاجل), normal (عادي), low (منخفض)

- enum CaseStage:
  received (استلام من العميل)
  assigned_to_department (محالة للقسم)
  collecting_documents (استكمال المستندات)
  under_study (دراسة القضية)
  drafting_lawsuit (تحرير الدعوى)
  drafting_response (كتابة المذكرة الجوابية)
  in_review (لدى لجنة المراجعة)
  review_notes_received (استلام ملاحظات المراجعة)
  processing_notes (معالجة الملاحظات)
  returned_for_revision (مُرجعة للتعديل)
  ready_to_submit (جاهزة للرفع)
  submitted_to_court (مرفوعة في المحكمة)

- enum ConsultationStage:
  received (استلام من العميل)
  assigned_to_department (محالة للقسم)
  drafting (تحرير الاستشارة)
  in_review (لدى لجنة المراجعة)
  review_notes_received (استلام ملاحظات المراجعة)
  processing_notes (معالجة الملاحظات)
  returned_for_revision (مُرجعة للتعديل)
  ready_to_send (جاهزة للإرسال)
  sent_to_client (مرسلة للعميل)

- enum ReviewNoteAction: 
  fully_accepted (الأخذ بها كلياً)
  partially_accepted (الأخذ بها جزئياً)
  rejected (عدم الأخذ بها)
  returned (مرفوضة - تحتاج إعادة)

- interface StageSLA:
  stage: CaseStage | ConsultationStage
  maxDurationHours: number
  warningBeforeHours: number

- interface StageTransition:
  id
  entityType: "case" | "consultation"
  entityId
  fromStage
  toStage
  performedBy: userId
  performedByRole: userRole
  notes: string
  duration: number (المدة بالساعات في المرحلة السابقة)
  isOverdue: boolean (هل تجاوز المدة المحددة)
  createdAt

- interface ReviewNote:
  id
  entityType: "case" | "consultation"
  entityId
  reviewerId: userId
  reviewerName
  notes: string
  action: ReviewNoteAction | null
  actionJustification: string
  acceptedItems: string[]
  rejectedItems: string[]
  returnCount: number (عدد مرات الإرجاع)
  returnReason: string (سبب الإرجاع إذا كان returned)
  createdAt
  respondedAt

- interface EmployeeWorkload:
  id
  name
  department
  activeCases: number
  activeConsultations: number
  inReviewItems: number
  overdueItems: number
  avgCompletionDays: number

- في interface Case أضف:
  clientRole: ClientRole
  priority: CasePriority
  currentStage: CaseStage
  stageStartedAt: Date (بداية المرحلة الحالية)
  stageDeadline: Date (موعد نهاية المرحلة)
  stageHistory: StageTransition[]
  reviewNotes: ReviewNote[]
  returnCount: number (عدد مرات الإرجاع الكلي)
  assignedDepartment: string
  assignedTo: userId (الموظف المسؤول)
  assignedBy: userId
  assignedAt: Date
  isOverdue: boolean

- في interface Consultation نفس الإضافات

=== 2. أنشئ client/src/lib/workflow-context.tsx ===

إعدادات SLA الافتراضية:
- استلام → إحالة للقسم: 4 ساعات
- استكمال المستندات: 48 ساعة
- دراسة القضية: 72 ساعة
- تحرير الدعوى/المذكرة: 48 ساعة
- المراجعة: 24 ساعة
- معالجة الملاحظات: 24 ساعة
- الرفع: 4 ساعات

دوال إدارة سلسلة العمل:

أساسية:
- receiveCaseFromClient(caseData, priority): إنشاء قضية مع الأولوية
- assignCaseToDepartment(caseId, departmentId, assignedTo): إحالة للقسم وتعيين موظف
- updateCaseStage(caseId, newStage, notes): تغيير المرحلة مع حساب المدة
- sendCaseToReview(caseId): إرسال للجنة المراجعة
- submitReviewNotes(caseId, notes, action): إضافة ملاحظات اللجنة
- respondToReviewNotes(caseId, noteId, action, justification): رد القسم
- returnCase(caseId, reason): إرجاع القضية للتعديل
- submitToCourtSystem(caseId): الرفع في المحكمة

SLA والوقت:
- checkOverdueItems(): فحص العناصر المتأخرة
- getStageDeadline(stage, priority): حساب الموعد النهائي
- getTimeRemaining(entityId): الوقت المتبقي
- getSLAReport(dateRange): تقرير الالتزام بالمواعيد

توازن العمل:
- getEmployeeWorkload(employeeId): حمل موظف
- getDepartmentWorkload(departmentId): حمل قسم
- getWorkloadOverview(): نظرة عامة على الأحمال
- suggestReassignment(caseId): اقتراح إعادة توزيع
- reassignCase(caseId, newAssignee): إعادة تعيين

مؤشرات الأداء:
- getReviewAcceptanceRate(employeeId | departmentId): نسبة قبول الملاحظات
- getAvgCompletionTime(entityType, stage): متوسط وقت الإنجاز
- getReturnRate(employeeId | departmentId): نسبة الإرجاع
- getProductivityReport(dateRange): تقرير الإنتاجية
- getBottleneckReport(): تقرير الاختناقات

=== 13. الإشعارات التلقائية ===

SLA:
- تنبيه قبل انتهاء مدة المرحلة بـ 20%
- تنبيه فوري عند التأخر
- تنبيه للمدير إذا تأخر أكثر من 24 ساعة إضافية

الإرجاع:
- إشعار للموظف عند إرجاع عمله
- إشعار للمدير إذا كان الإرجاع للمرة الثالثة
- إشعار أسبوعي بملخص الإرجاعات

توازن العمل:
- تنبيه للمدير إذا موظف تجاوز 15 قضية نشطة
- اقتراح إعادة توزيع تلقائي
