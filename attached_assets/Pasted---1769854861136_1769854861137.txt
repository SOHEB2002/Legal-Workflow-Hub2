# أمر تطوير نظام إدارة مكتب المحاماة - شركة العون

## المطلوب: إضافة الميزات التالية للنظام الحالي

---

## 1. شريط مراحل القضية (Case Progress Bar)

أضف شريط تقدم مرئي في صفحة تفاصيل القضية يعرض المراحل السبع:

```
المراحل بالترتيب:
1. استلام
2. استكمال البيانات
3. دراسة
4. تحرير المذكرة
5. إحالة للجنة المراجعة
6. الأخذ بالملاحظات
7. رفع للدائرة
```

**المتطلبات:**
- شريط أفقي يعرض جميع المراحل
- المرحلة الحالية مميزة باللون الذهبي (#d4af37)
- المراحل المكتملة باللون الأخضر
- المراحل القادمة باللون الرمادي
- زر "نقل للمرحلة التالية" مع تأكيد
- زر "إرجاع للمرحلة السابقة" (للمدير فقط)
- حفظ تاريخ ووقت كل انتقال مع اسم المستخدم

---

## 2. نظام المستندات (Documents System)

أضف نظام إدارة مستندات متكامل:

**جدول documents:**
```sql
- id
- name (اسم المستند)
- type (نوع: عقد، مذكرة، حكم، توكيل، مستند رسمي، أخرى)
- file_url (رابط الملف)
- file_size
- uploaded_by (المستخدم)
- uploaded_at
- case_id (مرتبط بقضية - اختياري)
- consultation_id (مرتبط باستشارة - اختياري)
- hearing_id (مرتبط بجلسة - اختياري)
- client_id (مرتبط بعميل - اختياري)
- classification (تصنيف: سري، داخلي، عام)
- notes
```

**الواجهة:**
- زر رفع مستندات في كل صفحة (قضية، استشارة، جلسة، عميل)
- قائمة المستندات المرتبطة
- معاينة المستندات (PDF, صور)
- تحميل المستندات
- أيقونات حسب نوع الملف

---

## 3. نظام الإشعارات (Notifications System)

**جدول notifications:**
```sql
- id
- user_id (المستلم)
- title
- message
- type (جلسة، مهمة، قضية، نظام)
- is_read (تم القراءة)
- link (رابط للصفحة المعنية)
- created_at
- send_whatsapp (إرسال واتساب)
- whatsapp_sent (تم الإرسال)
```

**المتطلبات:**
- أيقونة جرس في الـ Header مع عداد
- قائمة منسدلة بالإشعارات الأخيرة
- صفحة جميع الإشعارات
- إشعار تلقائي قبل الجلسة بـ 24 ساعة
- إشعار عند إسناد مهمة
- إشعار عند تغيير حالة القضية
- إشعار عند إضافة تعليق

**إشعارات واتساب (اختياري):**
- حقل رقم واتساب في بيانات المستخدم
- إرسال تذكير الجلسات عبر واتساب

---

## 4. صفحة التقارير (Reports Page)

أضف صفحة تقارير جديدة في القائمة الجانبية:

**التقارير المطلوبة:**

1. **تقرير القضايا:**
   - عدد القضايا حسب الحالة
   - عدد القضايا حسب النوع
   - عدد القضايا حسب القسم
   - القضايا المتأخرة

2. **تقرير الجلسات:**
   - جلسات الأسبوع القادم
   - جلسات الشهر
   - الجلسات حسب المحكمة

3. **تقرير الموظفين:**
   - عدد القضايا لكل موظف
   - المهام المكتملة والمعلقة
   - الأداء الشهري

4. **تقرير العملاء:**
   - العملاء الجدد
   - القضايا لكل عميل

**المتطلبات:**
- فلترة حسب التاريخ (من - إلى)
- تصدير PDF
- تصدير Excel
- رسوم بيانية (Charts) باستخدام Recharts

---

## 5. سجل المراجعة (Audit Log)

**جدول audit_logs:**
```sql
- id
- user_id (من قام بالعملية)
- user_name
- action (إضافة، تعديل، حذف، عرض، تسجيل دخول)
- entity_type (قضية، عميل، جلسة، مستخدم، إلخ)
- entity_id
- entity_name
- old_value (القيمة القديمة - JSON)
- new_value (القيمة الجديدة - JSON)
- ip_address
- created_at
```

**الواجهة:**
- صفحة جديدة "سجل العمليات" تحت الإدارة
- جدول يعرض جميع العمليات
- فلترة حسب: المستخدم، نوع العملية، التاريخ، نوع الكيان
- بحث
- الوصول لمدير الفرع فقط

---

## 6. نظام التفويض المؤقت (Temporary Delegation)

**جدول delegations:**
```sql
- id
- from_user_id (المفوِّض)
- to_user_id (المفوَّض إليه)
- start_date
- end_date
- reason (السبب: إجازة، مرض، سفر، أخرى)
- permissions (الصلاحيات المفوضة - JSON)
- is_active
- created_by
- created_at
```

**الواجهة:**
- زر "تفويض صلاحياتي" في صفحة الملف الشخصي
- نموذج: اختيار المستخدم، تاريخ البداية والنهاية، السبب
- قائمة التفويضات النشطة
- إلغاء التفويض
- إشعار للمفوَّض إليه

---

## 7. تحسينات صفحة القضايا

**إضافة في جدول القضايا:**
- عمود "المحامي المسؤول"
- عمود "المرحلة الحالية"

**إضافة في تفاصيل القضية:**
- رقم ناجز
- رقم القضية بالمحكمة
- المحامي المسؤول
- تبويب "الجلسات" يعرض جلسات هذه القضية
- تبويب "المستندات"
- تبويب "سجل التعديلات"
- تبويب "التعليقات"

---

## 8. تحسينات صفحة العملاء

**إضافة حقول:**
- العنوان
- تاريخ التسجيل (يظهر في الجدول)
- الملاحظات

**إضافة في تفاصيل العميل:**
- قائمة القضايا المرتبطة
- قائمة الاستشارات المرتبطة
- المستندات المرتبطة

---

## 9. نظام الصلاحيات المتقدم (RBAC)

**تحديث جدول roles:**
```sql
- id
- name
- permissions (JSON array)
- created_at
```

**الصلاحيات:**
```javascript
permissions = {
  cases: { view: true, create: true, edit: true, delete: false, change_status: true },
  consultations: { view: true, create: true, edit: true, delete: false },
  clients: { view: true, create: true, edit: true, delete: false },
  hearings: { view: true, create: true, edit: true, delete: false },
  field_tasks: { view: true, create: true, edit: true, delete: false },
  users: { view: false, create: false, edit: false, delete: false },
  reports: { view: true, export: true },
  audit_log: { view: false },
  settings: { view: false, edit: false }
}
```

**صفحة إدارة الصلاحيات:**
- قائمة الأدوار
- تعديل صلاحيات كل دور
- Checkboxes لكل صلاحية

---

## 10. التقويم (Calendar View)

**إضافة صفحة التقويم:**
- عرض شهري للجلسات والمهام
- ألوان مختلفة: جلسات (أزرق)، مهام ميدانية (أخضر)، مهام عاجلة (أحمر)
- النقر على اليوم يعرض التفاصيل
- إضافة جلسة/مهمة من التقويم مباشرة

استخدم مكتبة: react-big-calendar أو FullCalendar

---

## 11. إرفاق مستندات للجلسات

في صفحة الجلسات:
- زر "إرفاق مستند" بجانب كل جلسة
- عرض عدد المستندات المرفقة
- في تفاصيل الجلسة: قائمة المستندات مع إمكانية التحميل

---

## 12. تحسينات عامة

1. **Loading States:** إضافة Skeleton loaders أثناء التحميل
2. **Error Handling:** رسائل خطأ واضحة بالعربي
3. **Success Messages:** رسائل نجاح عند الحفظ
4. **Confirmation Dialogs:** تأكيد قبل الحذف
5. **Responsive Design:** تحسين للموبايل
6. **Dark Mode:** (اختياري) وضع داكن
