# أمر Replit Agent: تحسين دعم اللغة العربية والتوافق مع النصوص ثنائية الاتجاه

## انسخ النص التالي وألصقه في Replit Agent:

---

أحتاج تحسين دعم اللغة العربية وضمان عدم التعارض مع النصوص الإنجليزية أو الأرقام. النظام حالياً يعمل بالعربية بشكل جيد، لكن يحتاج تحسينات في التعامل مع النصوص المختلطة (أرقام لاتينية داخل نص عربي) وبعض التحسينات الأخرى.

مهم جداً: لا تغيّر أي وظيفة موجودة. فقط حسّن العرض والتوافق.

---

## ١. إصلاح اتجاه النصوص المختلطة (Bidi)

### المشكلة:
عندما يحتوي حقل عربي على أرقام أو نصوص إنجليزية (مثل: رقم قضية "1446/ت/123" أو اسم محكمة "المحكمة التجارية بالرياض Circuit 3") قد يظهر الاتجاه مقلوباً أو مشوّهاً.

### الحل - إنشاء مكون `BidiText`:

أنشئ ملف `client/src/components/ui/bidi-text.tsx`:
```typescript
import React from "react";

interface BidiTextProps {
  children: React.ReactNode;
  className?: string;
  as?: "span" | "p" | "div";
}

/**
 * مكون لعرض النصوص ثنائية الاتجاه بشكل صحيح
 * يضمن أن الأرقام اللاتينية والنصوص الإنجليزية تظهر بالاتجاه الصحيح داخل النص العربي
 */
export function BidiText({ children, className = "", as: Tag = "span" }: BidiTextProps) {
  return (
    <Tag
      className={className}
      style={{ unicodeBidi: "plaintext", direction: "rtl" }}
    >
      {children}
    </Tag>
  );
}

/**
 * مكون لعرض الأرقام والرموز بشكل LTR داخل سياق RTL
 * مفيد لأرقام القضايا والتواريخ بالتنسيق اللاتيني والأكواد
 */
export function LtrInline({ children, className = "" }: { children: React.ReactNode; className?: string }) {
  return (
    <span className={className} dir="ltr" style={{ unicodeBidi: "embed", display: "inline-block" }}>
      {children}
    </span>
  );
}
```

### تطبيق BidiText في الأماكن المطلوبة:

#### أ. في صفحة القضايا `client/src/pages/cases.tsx`:
استبدل عرض رقم القضية:
```typescript

```

استبدل عرض اسم العميل (قد يحتوي أسماء أجنبية):
```typescript

</TableCell>
```

#### ب. في صفحة الجلسات `client/src/pages/hearings.tsx`:
```typescript

// اسم المحكمة (قد يحتوي نص مختلط)
```

#### ج. في صفحة المذكرات `client/src/pages/memos.tsx`:
```typescript

```

#### د. في أي مكان يعرض:
- أرقام القضايا → `<LtrInline>`
- أرقام الهواتف → `<LtrInline>`

- أرقام الهوية → `<LtrInline>`
- أسماء قد تكون أجنبية → `<BidiText>`
- عناوين URL أو روابط → `<LtrInline>`

ابحث في جميع صفحات client/src/pages/ وكل مكان يعرض هذه البيانات وطبّق المكون المناسب.

## ٣. تحسين حقول الإدخال لدعم الكتابة المختلطة

### المشكلة:
عندما يكتب المستخدم في حقل بحث أو حقل نص ويخلط بين عربي وإنجليزي (مثلاً: رقم قضية يحتوي أرقام وحروف) قد يقفز المؤشر أو يتغير الاتجاه بشكل مربك.

### الحل:
أنشئ مكون `SmartInput` في `client/src/components/ui/smart-input.tsx`:
```typescript
import React, { useCallback } from "react";
import { Input } from "@/components/ui/input";

interface SmartInputProps extends React.InputHTMLAttributes<HTMLInputElement> {
  inputType?: "text" | "numeric" | "email" | "phone" | "code";
}

/**
 * حقل إدخال ذكي يتعامل مع اتجاه النص تلقائياً
 * - text: يكتشف اللغة تلقائياً ويضبط الاتجاه
 * - numeric: دائماً LTR (للأرقام وأرقام القضايا)
 * - email: دائماً LTR
 * - phone: دائماً LTR
 * - code: دائماً LTR (للأكواد والمعرفات)
 */
export function SmartInput({ inputType = "text", className = "", ...props }: SmartInputProps) {
  const getDirection = useCallback(() => {
    switch (inputType) {
      case "numeric":
      case "email":
      case "phone":
      case "code":
        return "ltr";
      default:
        return undefined; // يستخدم auto-detect
    }
  }, [inputType]);

  const getTextAlign = useCallback(() => {
    switch (inputType) {
      case "numeric":
      case "email":
      case "phone":
      case "code":
        return "left" as const;
      default:
        return "right" as const;
    }
  }, [inputType]);

  return (
    <Input
      {...props}
      dir={getDirection()}
      className={`${className}`}
      style={{
        ...props.style,
        textAlign: getTextAlign(),
        unicodeBidi: inputType === "text" ? "plaintext" : "embed",
      }}
    />
  );
}
```

### تطبيق SmartInput في النماذج:

## ٤. تحسين عرض التواريخ

### المشكلة:
التواريخ أحياناً تظهر بالتنسيق اللاتيني (2024-01-15) وأحياناً بالعربي. يجب توحيدها.

### الحل - إنشاء دالة تنسيق موحدة:
أنشئ ملف `client/src/lib/date-utils.ts`:
```typescript
import { format, formatDistanceToNow, isValid, parseISO } from "date-fns";
import { ar } from "date-fns/locale";

/**
 * تنسيق التاريخ بالعربية
 * @param date - التاريخ (string أو Date)
 * @param pattern - نمط التنسيق (افتراضي: "dd MMMM yyyy")
 */
export function formatDateAr(date: string | Date | null | undefined, pattern = "dd MMMM yyyy"): string {
  if (!date) return "—";
  try {
    const d = typeof date === "string" ? parseISO(date) : date;
    if (!isValid(d)) return String(date); // إرجاع النص كما هو إذا لم يكن تاريخ صالح
    return format(d, pattern, { locale: ar });
  } catch {
    return String(date);
  }
}

/**
 * تنسيق التاريخ والوقت بالعربية
 */
export function formatDateTimeAr(date: string | Date | null | undefined): string {
  return formatDateAr(date, "dd MMMM yyyy - HH:mm");
}

/**
 * تنسيق التاريخ المختصر
 */
export function formatDateShortAr(date: string | Date | null | undefined): string {
  return formatDateAr(date, "dd/MM/yyyy");
}

/**
 * "منذ X ساعات" أو "منذ 3 أيام" بالعربية
 */
export function timeAgoAr(date: string | Date | null | undefined): string {
  if (!date) return "—";
  try {
    const d = typeof date === "string" ? parseISO(date) : date;
    if (!isValid(d)) return String(date);
    return formatDistanceToNow(d, { locale: ar, addSuffix: true });
  } catch {
    return String(date);
  }
}

/**
 * تنسيق الوقت فقط
 */
export function formatTimeAr(time: string | null | undefined): string {
  if (!time) return "—";
  // تحويل "14:30" إلى "02:30 م"
  try {
    const [hours, minutes] = time.split(":").map(Number);
    const period = hours >= 12 ? "م" : "ص";
    const h = hours % 12 || 12;
    return `${h}:${String(minutes).padStart(2, "0")} ${period}`;
  } catch {
    return time;
  }
}

/**
 * تنسيق التاريخ الهجري (إذا أردت دعمه مستقبلاً)
 * حالياً يُرجع التاريخ الميلادي
 */
export function formatHijriDate(date: string | Date | null | undefined): string {
  if (!date) return "—";
  try {
    const d = typeof date === "string" ? new Date(date) : date;
    // استخدام Intl API لتحويل التاريخ للهجري
    return new Intl.DateTimeFormat("ar-SA-u-ca-islamic", {
      day: "numeric",
      month: "long",
      year: "numeric",
    }).format(d);
  } catch {
    return formatDateAr(date);
  }
}
```

### استبدال جميع تنسيقات التواريخ:
في جميع صفحات client/src/pages/:

## ٥. تحسين عرض الأرقام

### المشكلة:
الأرقام تظهر بالشكل اللاتيني (1, 2, 3) في كل مكان. بعض المستخدمين يفضلون الأرقام العربية (١، ٢، ٣) أو الهندية.

### الحل - دالة تنسيق الأرقام:
أضف في `client/src/lib/date-utils.ts` (أو ملف جديد `format-utils.ts`):
```typescript
/**
 * تنسيق الأرقام بالعربية
 * مثال: 1234 → "1,234" أو "١٬٢٣٤"
 */
export function formatNumberAr(num: number | string | null | undefined, useArabicDigits = false): string {
  if (num === null || num === undefined) return "—";
  const n = typeof num === "string" ? parseFloat(num) : num;
  if (isNaN(n)) return String(num);

  if (useArabicDigits) {
    return new Intl.NumberFormat("ar-SA").format(n);
  }
  // أرقام لاتينية مع فاصلة آلاف
  return new Intl.NumberFormat("en-US").format(n);
}

/**
 * تنسيق النسبة المئوية
 * مثال: 0.75 → "75%"
 */
export function formatPercentAr(value: number | null | undefined): string {
  if (value === null || value === undefined) return "—";
  return `${Math.round(value * 100)}%`;
}
```

## ٦. تحسين رسائل الخطأ والإشعارات

### المشكلة:
رسائل الخطأ من الخادم كلها بالعربي (جيد)، لكن بعض أخطاء الشبكة تظهر بالإنجليزي (مثل "Network Error" أو "Request failed").

### الحل:
أنشئ ملف `client/src/lib/error-messages.ts`:
```typescript
/**
 * ترجمة رسائل الخطأ الشائعة
 */
const errorTranslations: Record<string, string> = {
  "Network Error": "خطأ في الاتصال بالشبكة. تحقق من اتصالك بالإنترنت.",
  "Request failed with status code 401": "جلستك انتهت. يرجى تسجيل الدخول مجدداً.",
  "Request failed with status code 403": "لا تملك صلاحية لهذا الإجراء.",
  "Request failed with status code 404": "العنصر المطلوب غير موجود.",
  "Request failed with status code 500": "حدث خطأ في الخادم. حاول مرة أخرى.",
  "Request failed with status code 429": "طلبات كثيرة. انتظر قليلاً ثم حاول مجدداً.",
  "timeout of": "انتهت مهلة الطلب. حاول مرة أخرى.",
  "Failed to fetch": "فشل الاتصال بالخادم.",
  "ERR_NETWORK": "لا يوجد اتصال بالإنترنت.",
  "ECONNREFUSED": "الخادم غير متاح حالياً.",
};

export function translateError(error: any): string {
  if (!error) return "حدث خطأ غير متوقع.";

  const message = error?.response?.data?.error
    || error?.response?.data?.message
    || error?.message
    || String(error);

  // إذا الرسالة بالعربي أصلاً، ارجعها كما هي
  if (/[\u0600-\u06FF]/.test(message)) return message;

  // ابحث عن ترجمة
  for (const [eng, arb] of Object.entries(errorTranslations)) {
    if (message.includes(eng)) return arb;
  }

  // رسالة افتراضية
  return "حدث خطأ. حاول مرة أخرى.";
}
```

```typescript
export function usePageTitle(title?: string) {
```
