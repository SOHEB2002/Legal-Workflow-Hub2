# أمر Replit Agent: تطوير وتحسين النظام - المرحلة التطويرية

## انسخ النص التالي وألصقه في Replit Agent:

---

أحتاج تطبيق التحسينات التالية على النظام. هذه تحسينات فقط ولا تغيّر هيكلية النظام الحالية. كل ميزة مستقلة تماماً.

مهم جداً: لا تحذف أي وظيفة موجودة. فقط أضف وحسّن.

## ١. لوحة أداء المحامين (تحسين صفحة KPIs الموجودة)

صفحة `kpis.tsx` الموجودة تعرض إحصائيات عامة وأداء الموظفين. أحتاج تحسينها بالآتي:

### قواعد الوصول والفلترة:
- **المحامي العادي (employee)**: يرى فقط إحصائيات القضايا الخاصة **بقسمه** (حسب departmentId)، لا يرى أداء محامين أقسام أخرى
- **رئيس القسم (department_head)**: يرى إحصائيات قسمه فقط مع تفاصيل أداء كل محامي في قسمه
- **الدعم الإداري (admin_support)**: يرى جميع الأقسام مع إمكانية الفلترة حسب القسم
- **رئيس لجنة المراجعة (cases_review_head)**: يرى جميع الأقسام مع إمكانية الفلترة
- **رئيس الفرع (branch_manager)**: يرى كل شيء مع فلاتر متقدمة

### إضافة بطاقات KPI جديدة لكل محامي:
```
بطاقة المحامي تعرض:
├── الاسم والقسم والصورة الرمزية
├── القضايا النشطة: [عدد]
├── القضايا المنجزة هذا الشهر: [عدد]
├── نسبة الإنجاز: [%] (منجزة / إجمالي)
├── الجلسات المحدّثة في وقتها: [%]  ← جديد
│   (عدد الجلسات التي سُجلت نتيجتها خلال 8 ساعات / إجمالي الجلسات)
├── متوسط وقت إنجاز المذكرات: [أيام] ← جديد
│   (الفرق بين تاريخ إنشاء المذكرة وتاريخ اكتمالها)
├── نسبة الأحكام لصالحنا: [%] ← جديد
│   (أحكام لصالحنا / إجمالي الأحكام)
├── المذكرات المتأخرة: [عدد] ← جديد
└── مؤشر الأداء الإجمالي: [نجوم من 5] ← جديد
    (محسوب من: نسبة الإنجاز × 0.3 + تحديث الجلسات × 0.25 + سرعة المذكرات × 0.25 + نسبة الفوز × 0.2)
```

### إضافة فلاتر:
- فلتر القسم (dropdown): جميع الأقسام / قسم محدد
- فلتر الفترة: هذا الشهر / الشهر الماضي / آخر 3 أشهر / هذه السنة / الكل
- فلتر نوع القضية: الكل / مدعي جديد / مدعي مقيّد / مدعى عليه

### دالة التقرير الأسبوعي:
```typescript
async function generateWeeklyReport() {
  const now = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 24 * 60 * 60 * 1000);

  // جمع البيانات
  const allCases = await storage.getCases();
  const allHearings = await storage.getHearings();
  const allMemos = await storage.getMemos();
  const allTickets = await storage.getSupportTickets();

  const newCases = allCases.filter(c => new Date(c.createdAt) >= weekAgo);
  const closedCases = allCases.filter(c => c.currentStage === "مغلقة" && new Date(c.updatedAt) >= weekAgo);

  const completedHearings = allHearings.filter(h =>
    h.status === "تمت" && new Date(h.updatedAt) >= weekAgo
  );
  const unupdatedHearings = allHearings.filter(h => {
    if (h.status !== "قادمة") return false;
    const hDate = new Date(h.hearingDate);
    return hDate < now && hDate >= weekAgo;
  });

  const overdueMemos = allMemos.filter(m => {
    if (m.status === "مكتمل" || m.status === "ملغي" || !m.deadline) return false;
    return new Date(m.deadline) < now;
  });

  const openTickets = allTickets.filter(t =>
    t.status !== "تم_الحل" && t.status !== "مغلقة"
  );

  // بناء نص التقرير
  const reportMessage = `
📊 التقرير الأسبوعي - ${now.toLocaleDateString("ar-SA")}

📁 القضايا:
• قضايا جديدة: ${newCases.length}
• قضايا مغلقة: ${closedCases.length}
• إجمالي القضايا النشطة: ${allCases.filter(c => c.currentStage !== "مغلقة" && c.currentStage !== "مؤرشفة").length}

⚖️ الجلسات:
• جلسات تمت هذا الأسبوع: ${completedHearings.length}
• جلسات فاتت بدون تحديث: ${unupdatedHearings.length}

📝 المذكرات:
• مذكرات متأخرة عن الموعد: ${overdueMemos.length}

🎫 تذاكر الدعم المفتوحة: ${openTickets.length}
  `.trim();

  // إرسال لمدير الفرع ورئيس لجنة المراجعة
  const managers = (await storage.getUsers()).filter(u =>
    u.role === "branch_manager" || u.role === "cases_review_head"
  );

  for (const manager of managers) {
    await storage.createNotification({
      type: "weekly_report",
      title: `📊 التقرير الأسبوعي - ${now.toLocaleDateString("ar-SA")}`,
      message: reportMessage,
      priority: "medium",
      status: "pending",
      senderId: "system",
      senderName: "النظام التلقائي",
      recipientId: manager.id,
    });
  }
}
```

أضف `"weekly_report"` في NotificationType في schema.ts.

النظام فيه جدول `attachments` بالفعل. المطلوب تحسينه:

### أ. تحسين واجهة رفع الملفات:
في صفحة تفاصيل القضية، أضف تبويب "المرفقات" يعرض:
- قائمة الملفات المرفقة بالقضية مع: اسم الملف، الحجم، النوع، من رفعه، التاريخ
- زر "رفع ملف" يفتح نافذة اختيار ملف
- إمكانية ربط المرفق بجلسة محددة أو مذكرة محددة (ليس فقط القضية)
- أيقونة حسب نوع الملف (PDF، Word، صورة، أخرى)

### ب. إعدادات الرفع في الباكند:
```typescript
// استخدم multer لرفع الملفات
import multer from "multer";

const upload = multer({
  storage: multer.diskStorage({
    destination: "./uploads",
    filename: (req, file, cb) => {
      const uniqueName = `${Date.now()}-${file.originalname}`;
      cb(null, uniqueName);
    }
  }),
  limits: { fileSize: 10 * 1024 * 1024 }, // 10MB حد أقصى
  fileFilter: (req, file, cb) => {
    const allowed = [
      "application/pdf",
      "application/msword",
      "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
      "image/jpeg", "image/png", "image/gif",
      "application/vnd.ms-excel",
      "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
    ];
    cb(null, allowed.includes(file.mimetype));
  }
});

// Endpoints
app.post("/api/attachments", requireAuth, upload.single("file"), async (req, res) => {
  // حفظ معلومات المرفق في قاعدة البيانات
  // تسجيل في سجل النشاط: attachment_added

app.get("/api/attachments/:id/download", requireAuth, async (req, res) => {
  // تحميل الملف
});

app.delete("/api/attachments/:id", requireAuth, requireRole("branch_manager", "department_head"), async (req, res) => {
  // حذف المرفق (مع تسجيل في سجل النشاط)
});
```

## ٥. نظام التوكيلات والصلاحيات المؤقتة

### أ. إنشاء جدول `delegations` في `shared/schema.ts`:
```typescript
export const delegations = pgTable("delegations", {
  id: varchar("id", { length: 255 }).primaryKey(),
  fromUserId: varchar("from_user_id", { length: 255 }).notNull(),
  toUserId: varchar("to_user_id", { length: 255 }).notNull(),

  // الأسباب: "إجازة", "مرض", "مهمة_خارجية", "تدريب", "أخرى"
  reasonDetails: text("reason_details"),

  endDate: varchar("end_date", { length: 50 }).notNull(),
  status: varchar("status", { length: 50 }).notNull().default("نشط"),
  // الحالات: "نشط", "منتهي", "ملغي"
  scope: varchar("scope", { length: 50 }).notNull().default("all_cases"),
  // النطاق: "all_cases" (كل القضايا), "specific_cases" (قضايا محددة)
  specificCaseIds: jsonb("specific_case_ids"), // مصفوفة أرقام قضايا (إذا كان النطاق محدد)
  approvedBy: varchar("approved_by", { length: 255 }),
  approvedAt: timestamp("approved_at"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

### ب. آلية العمل:
1. المحامي يطلب تفويض من صفحة "التفويضات" (صفحة جديدة)
2. يختار المحامي البديل + السبب + الفترة + القضايا (كلها أو محددة)
3. يُرسل إشعار لرئيس القسم للموافقة
4. بعد الموافقة: المحامي البديل يرى القضايا المفوَّض بها في لوحته
5. إشعار تلقائي للمحامي البديل: "تم تفويضك على قضايا [اسم] من [تاريخ] إلى [تاريخ]"
6. عند انتهاء الفترة: التفويض ينتهي تلقائياً + إشعار للطرفين
7. في المجدوِل scheduler.ts: فحص يومي لإنهاء التفويضات المنتهية

### ج. فحص المواعيد في المجدوِل:
```typescript
// في scheduler.ts - كل 6 ساعات
cron.schedule("0 */6 * * *", async () => {
  await checkLegalDeadlines();
});

async function checkLegalDeadlines() {
  const deadlines = await storage.getLegalDeadlines();
  const now = new Date();

  for (const deadline of deadlines) {
    if (deadline.status !== "نشط") continue;
    const dDate = new Date(deadline.deadlineDate);
    const daysLeft = (dDate.getTime() - now.getTime()) / (1000 * 60 * 60 * 24);

    // تنبيه قبل 7 أيام
    if (daysLeft <= 7 && daysLeft > 3 && !deadline.reminder7daysSent) {
      // إرسال إشعار بأولوية عالية
      // تحديث reminder7daysSent = true
    }
    // تنبيه قبل 3 أيام
```

```typescript
app.get("/api/stats/court-analytics", requireAuth, async (req, res) => {
  const cases = await storage.getCases();
  const hearings = await storage.getHearings();

  // توزيع القضايا حسب المحكمة
  const byCourtType = {};
  hearings.forEach(h => {
    const court = h.courtName || "غير محدد";
    byCourtType[court] = (byCourtType[court] || 0) + 1;
  });

  // نسبة الربح/الخسارة حسب نوع القضية
  const byResult = {};
  hearings.filter(h => h.result === "حكم").forEach(h => {
    const caseInfo = cases.find(c => c.id === h.caseId);
    const caseType = caseInfo?.caseType || "غير محدد";
    if (!byResult[caseType]) byResult[caseType] = { won: 0, lost: 0, partial: 0 };
    if (h.judgmentSide === "لصالحنا") byResult[caseType].won++;
    else if (h.judgmentSide === "ضدنا") byResult[caseType].lost++;
    else byResult[caseType].partial++;
  });

  // متوسط مدة القضايا حسب النوع
  const avgDuration = {};
  cases.filter(c => c.currentStage === "مغلقة").forEach(c => {
    const type = c.caseType || "غير محدد";
    const duration = (new Date(c.updatedAt).getTime() - new Date(c.createdAt).getTime()) / (1000 * 60 * 60 * 24);
    if (!avgDuration[type]) avgDuration[type] = [];
    avgDuration[type].push(duration);
  });

  res.json({ byCourtType, byResult, avgDuration });
});
```
