قم ببناء نظام إدارة متكامل لمكتب محاماة سعودي باللغة العربية.

## التقنيات المطلوبة:
- Frontend: React + TypeScript + Tailwind CSS
- Backend: Node.js + Express
- Database: Supabase (PostgreSQL + Auth + Realtime)
- Storage: Google Drive API للمستندات
- Notifications: واتساب (عبر WhatsApp Business API أو Twilio) + Email (SendGrid/Resend)
- Calendar: Google Calendar API
- Hosting: Railway أو Render

## التصميم:
- اللغة: عربية بالكامل (RTL)
- الثيم: كحلي داكن (#1a365d) + ذهبي (#d4af37) + أبيض
- خط: Cairo أو Tajawal
- تصميم متجاوب (Mobile-first) للتحضير لتطبيق جوال مستقبلاً
- وضع ليلي (Dark Mode)

## الهيكل التنظيمي والصلاحيات (RBAC):

### المستوى 1 - الإدارة العليا:

**1. مدير الفرع**
- وصول كامل لكل شيء
- إدارة المستخدمين والصلاحيات
- جميع التقارير
- التفويض المؤقت

**2. رئيس لجنة مراجعة القضايا**
- وصول كامل لجميع القضايا من كل الأقسام
- اعتماد/إعادة القضايا مع ملاحظات
- تقارير القضايا

**3. رئيس لجنة مراجعة الاستشارات**
- وصول كامل لجميع الاستشارات من كل الأقسام
- اعتماد/إعادة الاستشارات مع ملاحظات
- تقارير الاستشارات

### المستوى 2 - رؤساء الأقسام:

**4. رئيس القسم العام**
**5. رئيس القسم التجاري**
**6. رئيس القسم العمالي**
**7. رئيس القسم الإداري**

صلاحيات رؤساء الأقسام:
- إدارة قضايا واستشارات قسمه فقط
- توزيع العمل على موظفي القسم
- مراجعة أولية قبل إرسال للجنة
- تقارير القسم
- التفويض لموظف في غيابه

### المستوى 3 - الدعم:

**8. الدعم الإداري**
- إضافة قضايا واستشارات جديدة
- استقبال الطلبات وتوجيهها للأقسام
- رؤية جميع القضايا (بدون تعديل المحتوى القانوني)
- إغلاق وأرشفة الملفات الجاهزة
- إضافة بيانات العملاء

### المستوى 4 - الموظفين:

**9. موظف قسم**
- رؤية والعمل على القضايا المسندة إليه فقط
- إضافة ملاحظات ومذكرات
- رفع مستندات
- إرسال لرئيس القسم للمراجعة

### المستوى 5 - الموارد البشرية:

**10. موظف الموارد البشرية**
- إدارة بيانات الموظفين الشخصية
- الإجازات والحضور والانصراف
- لا وصول للقضايا والاستشارات

## دورة عمل القضايا (Workflow):

[الدعم الإداري] إضافة قضية ← الحالة: "استلام"
         ↓
[الدعم الإداري] توجيه للقسم المختص ← الحالة: "استكمال البيانات"
         ↓
[رئيس القسم] إسناد لمحامي (أو أكثر) ← الحالة: "دراسة"
         ↓
[المحامي] دراسة وتحرير المذكرة ← الحالة: "تحرير المذكرة"
         ↓
[المحامي] إرسال لرئيس القسم
         ↓
[رئيس القسم] مراجعة أولية ← إرسال للجنة المراجعة ← الحالة: "لجنة المراجعة"
         ↓
[رئيس لجنة مراجعة القضايا] المراجعة:
    ├── اعتماد ← الحالة: "جاهز للرفع"
    ├── إعادة كلية ← الحالة: "تعديلات" (تعود للمحامي)
    └── اعتماد جزئي مع ملاحظات ← الحالة: "تعديلات"
         ↓
[المحامي بعد التعديل]:
    ├── الأخذ بالملاحظات ← إعادة إرسال للجنة
    ├── عدم الأخذ (مع تبرير) ← إعادة إرسال للجنة
    └── الأخذ الجزئي ← إعادة إرسال للجنة
         ↓
[بعد الاعتماد النهائي] ← الحالة: "جاهز للرفع"
         ↓
[المحامي/رئيس القسم] رفع للمحكمة ← الحالة: "مرفوع"
         ↓
[عند انتهاء القضية] ← الحالة: "مغلق"

---

## دورة عمل الاستشارات:

[الدعم الإداري] إضافة استشارة ← "استلام"
         ↓
[الدعم الإداري] توجيه للقسم ← "دراسة"
         ↓
[رئيس القسم] إسناد لمحامي
         ↓
[المحامي] إعداد الرد ← "إعداد الرد"
         ↓
[رئيس القسم] مراجعة ← إرسال للجنة ← "لجنة المراجعة"
         ↓
[رئيس لجنة مراجعة الاستشارات]:
    ├── اعتماد ← "جاهز"
    └── إعادة ← "تعديلات"
         ↓
[بعد الاعتماد] ← "جاهز"
         ↓
[الدعم الإداري] تسليم للعميل ← "مسلّم"
         ↓
[إذا تحولت لقضية] ← إنشاء قضية مرتبطة
         ↓
[إغلاق] ← "مغلق"

## الشاشات المطلوبة:

### 1. تسجيل الدخول:
- تصميم فخم مع شعار المكتب
- البريد + كلمة المرور
- تذكرني
- نسيت كلمة المرور

### 2. لوحات التحكم (Dashboard) - مختلفة حسب الدور:

**مدير الفرع:**
- إحصائيات شاملة (قضايا، استشارات، حسب القسم، حسب الحالة)
- رسم بياني للأداء الشهري
- أحدث النشاطات
- تنبيهات مهمة
- الجلسات القادمة (الكل)

**رئيس لجنة المراجعة (قضايا/استشارات):**
- القضايا/الاستشارات بانتظار المراجعة (بارز)
- إحصائيات المراجعة (معتمد، معاد، جزئي)
- الجلسات القادمة

**رئيس القسم:**
- إحصائيات قسمه فقط
- القضايا قيد العمل في القسم
- توزيع العمل على الموظفين
- الجلسات القادمة للقسم

**الدعم الإداري:**
- الطلبات الجديدة (للتوجيه)
- الجاهزة للتسليم
- إضافة سريعة (قضية/استشارة/عميل)
- الجلسات القادمة

**الموظف:**
- المهام المسندة إليه
- حالة كل قضية/استشارة
- الجلسات القادمة له
- إحصائيات أدائه الشخصي

**الموارد البشرية:**
- إحصائيات الموظفين
- طلبات الإجازات
- سجل الحضور

### 3. إدارة القضايا:
- جدول مع فلاتر متقدمة (القسم، الحالة، النوع، المحامي، الفترة، الأولوية)
- بحث (رقم القضية، اسم العميل، رقم ناجز)
- أزرار الإجراءات حسب الصلاحية والحالة
- تصدير Excel

**صفحة تفاصيل القضية:**
- معلومات العميل
- معلومات القضية والمحكمة
- الخصم ومحاميه
- المحامين المسندين
- Timeline كامل للإجراءات
- المستندات (مع رفع لـ Google Drive)
- الجلسات المرتبطة
- ملاحظات المراجعة والردود
- روابط (واتساب، درايف)

### 4. إدارة الاستشارات:
- نفس هيكل القضايا
- زر "تحويل لقضية"

### 5. إدارة الجلسات:
- عرض تقويم (شهري/أسبوعي/يومي)
- قائمة الجلسات القادمة
- تنبيهات لونية:
  - أحمر: اليوم
  - برتقالي: خلال 3 أيام
  - أصفر: خلال أسبوع
- مزامنة مع Google Calendar
- إضافة/تعديل جلسة

### 6. إدارة العملاء:
- قائمة العملاء (أفراد + شركات)
- بحث وفلترة
- صفحة تفاصيل العميل مع قضاياه واستشاراته

### 7. بوابة العملاء (Client Portal):
- تسجيل دخول منفصل
- عرض قضاياه وحالتها
- عرض استشاراته
- الجلسات القادمة
- تحميل المستندات الخاصة به
- لا يمكنه التعديل

### 8. إدارة المستخدمين (مدير الفرع فقط):
- إضافة/تعديل/تعطيل مستخدم
- تحديد الدور والقسم
- الصلاحيات التفصيلية
- التفويضات المؤقتة

### 9. الموارد البشرية:
- بيانات الموظفين
- سجل الحضور والانصراف
- طلبات الإجازات والموافقة عليها

### 10. التقارير:
- تقرير القضايا (حسب القسم، الحالة، الفترة، المحامي)
- تقرير الاستشارات
- تقرير الجلسات الشهري
- تقرير أداء الموظفين
- تقرير المراجعات (نسبة القبول/الرفض)
- تصدير PDF/Excel

### 11. سجل النشاط (Audit Log):
- كل العمليات مسجلة
- فلترة حسب المستخدم، التاريخ، نوع العملية
- عرض القيمة القديمة والجديدة

### 12. الإعدادات:
- إعدادات الحساب الشخصي
- إعدادات الإشعارات
- إعدادات النظام (مدير فقط)
- إدارة المحاكم (إضافة محاكم جديدة)
