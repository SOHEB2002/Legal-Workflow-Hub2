# أمر Replit Agent: إصلاح الثغرات الأمنية الحرجة وتحسين جودة الكود

## انسخ النص التالي وألصقه في Replit Agent:

---

أحتاج إصلاح ثغرات أمنية حرجة وتحسينات في جودة الكود قبل نشر النظام. هذه التعديلات ضرورية جداً ولا يمكن تأجيلها:

### أولاً: إصلاح مفتاح JWT (حرج جداً)

1. في `server/auth.ts`:
   - احذف القيمة الافتراضية "oun-law-jwt-secret-2024" من `JWT_SECRET`
   - اجعل المتغير يقرأ فقط من `process.env.SESSION_SECRET`
   - إذا كان المتغير غير موجود، اطبع رسالة خطأ وأوقف السيرفر:
     ```typescript
     const JWT_SECRET = process.env.SESSION_SECRET;
     if (!JWT_SECRET) {
       console.error("FATAL: SESSION_SECRET environment variable is required");
       process.exit(1);
     }
     ```
   - غيّر مدة صلاحية التوكن من `7d` إلى `2h` (ساعتين)
   - أضف آلية refresh token:
     * أنشئ endpoint جديد `POST /api/auth/refresh`
     * يقبل التوكن الحالي (إذا لم تنتهِ صلاحيته بعد أو انتهت خلال آخر 30 دقيقة)
     * يُرجع توكن جديد بصلاحية ساعتين
   - في الفرونت إند `auth-context.tsx`:
     * أضف timer يجدد التوكن تلقائياً قبل انتهائه بـ 10 دقائق
     * إذا فشل التجديد، سجّل خروج المستخدم

### ثانياً: حماية جميع الـ API Endpoints (حرج جداً)

2. أنشئ middleware للتحقق من المصادقة في `server/auth.ts`:
   ```typescript
   export function requireAuth(req: Request, res: Response, next: NextFunction) {
     const token = req.headers.authorization?.replace("Bearer ", "");
     if (!token) return res.status(401).json({ error: "غير مصرح" });
     try {
       const decoded = jwt.verify(token, JWT_SECRET);
       req.user = decoded;
       next();
     } catch {
       return res.status(401).json({ error: "جلسة منتهية" });
     }
   }
   ```

3. أنشئ middleware للتحقق من الصلاحيات:
   ```typescript
   export function requireRole(...roles: string[]) {
     return (req: Request, res: Response, next: NextFunction) => {
       if (!req.user || !roles.includes(req.user.role)) {
         return res.status(403).json({ error: "لا تملك صلاحية" });
       }
       next();
     };
   }
   ```

4. في `server/routes.ts` أضف `requireAuth` على جميع الـ endpoints:
   - `app.get("/api/cases", requireAuth, ...)`
   - `app.post("/api/cases", requireAuth, ...)`
   - `app.patch("/api/cases/:id", requireAuth, ...)`
   - `app.delete("/api/cases/:id", requireAuth, requireRole("admin", "branch_manager"), ...)`
   - نفس الشيء لجميع endpoints: hearings, memos, users, teams, notifications, support, kpis, standards
   - endpoints الحذف تتطلب `requireRole("admin", "branch_manager")` إضافية
   - endpoints إدارة المستخدمين تتطلب `requireRole("admin", "branch_manager")`

### رابعاً: سياسة كلمات المرور (حرج)

7. في `server/auth.ts`:
   - أضف دالة التحقق من قوة كلمة المرور:
     ```typescript
     function validatePassword(password: string): { valid: boolean; message: string } {
       if (password.length < 8) return { valid: false, message: "كلمة المرور يجب أن تكون 8 أحرف على الأقل" };
       if (!/[A-Z]/.test(password) && !/[a-z]/.test(password))
         return { valid: false, message: "يجب أن تحتوي على حروف" };
       if (!/[0-9]/.test(password)) return { valid: false, message: "يجب أن تحتوي على أرقام" };
       return { valid: true, message: "" };
     }
     ```
   - طبّقها على endpoint تغيير كلمة المرور وإنشاء المستخدم
   - أضف حقل `mustChangePassword: boolean` في جدول users (default: true)
   - عند تسجيل الدخول، إذا `mustChangePassword = true`، أرسل في الاستجابة `{ mustChangePassword: true }`
   - في الفرونت إند: إذا كان `mustChangePassword = true`، اعرض نموذج تغيير كلمة المرور الإجباري قبل الوصول للنظام

8. غيّر كلمة المرور الافتراضية في `server/storage.ts`:
   - من "1234" إلى كلمة مرور عشوائية قوية (أو اجعلها متغير بيئة)
   - اجعل `mustChangePassword: true` للمستخدمين الافتراضيين

### خامساً: حماية من هجمات القوة الغاشمة (حرج)

9. أضف rate limiting على endpoint تسجيل الدخول:
   - استخدم `express-rate-limit`:
     ```bash
     npm install express-rate-limit
     ```
   - طبّقه على `/api/auth/login`:
     ```typescript
     import rateLimit from "express-rate-limit";

     const loginLimiter = rateLimit({
       windowMs: 15 * 60 * 1000, // 15 دقيقة
       max: 5, // 5 محاولات
       message: { error: "محاولات كثيرة. حاول بعد 15 دقيقة" },
       standardHeaders: true,
     });

     app.post("/api/auth/login", loginLimiter, ...);
     ```
   - أضف rate limiting عام على جميع API endpoints:
     ```typescript
     const apiLimiter = rateLimit({
       windowMs: 60 * 1000, // دقيقة واحدة
       max: 100, // 100 طلب في الدقيقة
     });
     app.use("/api/", apiLimiter);
     ```

10. أنشئ دالة لتنظيف بيانات المستخدم قبل الإرسال:

    function sanitizeUser(user: any) {
