# أمر Replit Agent: نظام الدعم الفني والاقتراحات

## الهدف
بناء نظام دعم فني متكامل داخل التطبيق يتيح لجميع المستخدمين (بكل الأدوار) إرسال بلاغات عن مشاكل تقنية أو اقتراحات تطوير، مع نظام تتبع ومتابعة وردود.

---

## القسم 1: جدول التذاكر

أنشئ جدول جديد في `shared/schema.ts` اسمه `support_tickets`:

```typescript
export const supportTickets = pgTable("support_tickets", {
  id: varchar("id", { length: 255 }).primaryKey(),
  ticketNumber: varchar("ticket_number", { length: 50 }).notNull().unique(),
  // نوع التذكرة
  ticketType: varchar("ticket_type", { length: 50 }).notNull(),
  // العنوان
  title: varchar("title", { length: 500 }).notNull(),
  // الوصف التفصيلي
  description: text("description").notNull(),
  // الصفحة أو المكان الذي حدثت فيه المشكلة
  relatedPage: varchar("related_page", { length: 255 }).default(""),
  // لقطة شاشة (رابط أو base64)
  screenshotUrl: text("screenshot_url").default(""),
  // الأولوية
  priority: varchar("priority", { length: 50 }).notNull().default("متوسط"),
  // الحالة
  status: varchar("status", { length: 50 }).notNull().default("جديدة"),
  // المُرسل
  submittedBy: varchar("submitted_by", { length: 255 }).notNull(),
  // المسؤول عن الحل (يُعيّن لاحقاً)
  assignedTo: varchar("assigned_to", { length: 255 }),
  // الردود والتعليقات
  comments: jsonb("comments").default([]),
  // التقييم بعد الإغلاق
  rating: integer("rating"),
  ratingComment: text("rating_comment").default(""),
  // تواريخ
  resolvedAt: timestamp("resolved_at"),
  closedAt: timestamp("closed_at"),
  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow(),
});
```

شغّل `drizzle-kit push`.

## القسم 3: صفحة الدعم الفني (للمستخدم العادي)

أنشئ صفحة `client/src/pages/support.tsx`:

### 3.1 واجهة المستخدم العادي (جميع الأدوار):

#### تبويبان رئيسيان:

**تبويب "تذاكري":**
- جدول يعرض التذاكر التي أرسلها المستخدم الحالي فقط
- أعمدة: رقم التذكرة | العنوان | النوع | الأولوية | الحالة | تاريخ الإنشاء | آخر تحديث
- تلوين الحالة:
  - جديدة: رمادي
  - مفتوحة: أزرق
  - قيد المعالجة: برتقالي
  - بانتظار رد: أصفر
  - تم الحل: أخضر
  - مغلقة: رمادي فاتح
- فلتر حسب الحالة والنوع
- عند الضغط على تذكرة: فتح تفاصيلها مع المحادثة

**تبويب "تذكرة جديدة":**
- نموذج إنشاء تذكرة جديدة

### 3.2 نموذج إنشاء تذكرة جديدة:

النموذج يكون واضح وبسيط:

```
┌──────────────────────────────────────────────────────────┐
│                   📝 تذكرة دعم جديدة                      │
│                                                          │
│  نوع التذكرة:                                             │
│  ┌──────────────────────────────────────────────────┐    │
│  │ [خلل فني] [اقتراح تطوير] [استفسار]              │    │
│  │ [مشكلة واجهة] [بطء أداء] [أخرى]                 │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  العنوان: [________________________________]              │
│                                                          │
│  الوصف التفصيلي:                                         │
│  ┌──────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  اشرح المشكلة أو الاقتراح بالتفصيل...           │    │
│  │                                                  │    │
│  └──────────────────────────────────────────────────┘    │
│                                                          │
│  الصفحة المتعلقة: [  القضايا ▼  ] (اختياري)              │
│                                                          │
│  الأولوية: [  متوسط ▼  ]                                  │
│                                                          │
│  لقطة شاشة: [ 📎 إرفاق صورة ] (اختياري)                  │
│                                                          │
│                              [ إرسال التذكرة ]            │
└──────────────────────────────────────────────────────────┘
```

**حقل "الصفحة المتعلقة":** قائمة منسدلة بأسماء صفحات النظام:
- لوحة التحكم
- إدارة القضايا
- الاستشارات
- الجلسات
- المذكرات
- المهام الميدانية
- الإشعارات
- المستخدمين
- أخرى

**حقل "نوع التذكرة":** يظهر كأزرار اختيار (toggle buttons) وليس قائمة منسدلة، كل زر بأيقونة مميزة:
- 🐛 خلل فني
- 💡 اقتراح تطوير
- ❓ استفسار
- 🖥️ مشكلة واجهة
- 🐢 بطء أداء
- 📋 أخرى

(استخدم أيقونات lucide-react بدلاً من الإيموجي)

### 3.3 تفاصيل التذكرة (Dialog):

عند فتح تذكرة:

```
┌──────────────────────────────────────────────────────────┐
│  تذكرة #TK-0042                         [حالة: مفتوحة]  │
│  ─────────────────────────────────────────────────────── │
│  النوع: خلل فني          الأولوية: عالي                  │
│  الصفحة: إدارة القضايا    المسؤول: أحمد (مدير الفرع)     │
│  ─────────────────────────────────────────────────────── │
│                                                          │
│  الوصف:                                                  │
│  عند محاولة تعديل القضية رقم 2024-001 يظهر خطأ...       │
│                                                          │
│  ─────────────────── المحادثة ──────────────────────      │
│                                                          │
│  [محمد - 10:30 ص]                                        │
│  المشكلة تظهر عند الضغط على زر الحفظ بعد...              │
│                                                          │
│  [أحمد (مدير) - 11:45 ص]                                 │
│  شكراً، جاري التحقق من المشكلة...                        │
│                                                          │
│  ┌──────────────────────────────────────────────────┐    │
│  │  اكتب ردك هنا...                                 │    │
│  └──────────────────────────────────────────────────┘    │
│  [ إرسال الرد ]                                          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- المستخدم العادي يرى تذكرته + يقدر يرد في المحادثة
- لا يرى الملاحظات الداخلية (isInternal = true)
- إذا الحالة "تم_الحل": يظهر زر "تأكيد الحل وإغلاق" + "المشكلة لا تزال قائمة" (يعيد فتحها)
- إذا الحالة "مغلقة": يظهر نموذج تقييم (1-5 نجوم + تعليق اختياري)

## القسم 4: دور الدعم الفني الجديد

### 4.1 إضافة دور "دعم فني" في النظام:

في `shared/schema.ts`، أضف دور جديد في `UserRole`:

```typescript

  BRANCH_MANAGER: "branch_manager",
  CASES_REVIEW_HEAD: "cases_review_head",
  CONSULTATIONS_REVIEW_HEAD: "consultations_review_head",
  DEPARTMENT_HEAD: "department_head",
  ADMIN_SUPPORT: "admin_support",
  EMPLOYEE: "employee",
  HR: "hr",
  TECHNICAL_SUPPORT: "technical_support",  // ← جديد: دعم فني
} as const;
```

أضف التسمية في `UserRoleLabels`:
```typescript
technical_support: "دعم فني",
```

أضف صلاحياته في `RolePermissions`:
```typescript
technical_support: [
  "view_cases",          // يرى القضايا (للتشخيص فقط)
  "view_consultations",  // يرى الاستشارات (للتشخيص فقط)
```

**ملاحظة مهمة:** دور الدعم الفني صلاحياته **للقراءة فقط** على بيانات النظام (ليتمكن من فهم المشاكل المُبلغ عنها)، لكن لديه **صلاحية كاملة** على نظام التذاكر.

## القسم 5: لوحة إدارة الدعم الفني (للمسؤولين)

### 5.1 من يرى لوحة الإدارة:

ثلاثة أدوار لديها صلاحيات إدارية على التذاكر:

| الصلاحية | مدير الفرع | رئيس لجنة مراجعة القضايا | دعم فني |
|----------|-----------|------------------------|---------|
| عرض كل التذاكر | ✅ | ✅ | ✅ |
| تعيين مسؤول | ✅ | ✅ | ✅ |
| تغيير الحالة | ✅ | ✅ | ✅ |
| تغيير الأولوية | ✅ | ✅ | ✅ |
| إضافة ملاحظة داخلية | ✅ | ✅ | ✅ |
| الرد على التذاكر | ✅ | ✅ | ✅ |
| إغلاق التذكرة | ✅ | ✅ | ✅ |
| حذف التذكرة | ✅ | ❌ | ❌ |
| عرض الإحصائيات | ✅ | ✅ | ✅ |

- **بقية الأدوار**: يرون فقط تذاكرهم الشخصية

### 5.2 كيف نتحقق من صلاحية الإدارة:

أضف دالة في `shared/schema.ts`:
```typescript
export function canManageSupportTickets(role: UserRoleType): boolean {
  return ["branch_manager", "cases_review_head", "technical_support"].includes(role);
}
```

### 5.3 واجهة المسؤولين (مدير الفرع + رئيس لجنة المراجعة + دعم فني):

أضف تبويبات إضافية في صفحة الدعم تظهر فقط لمن لديه `canManageSupportTickets`:

**تبويب "كل التذاكر":**
- جدول بكل تذاكر النظام
- أعمدة: الرقم | العنوان | النوع | المُرسل | الدور | الأولوية | الحالة | المسؤول | تاريخ الإنشاء
- فلاتر: الحالة، النوع، الأولوية، المُرسل
- ترتيب: الأحدث أولاً، مع إبراز التذاكر العاجلة في الأعلى

**تبويب "المعيّنة لي":**
- التذاكر المعيّنة للمستخدم الحالي (المسؤول عنها)
- مفيد عندما يكون هناك أكثر من شخص يتعامل مع التذاكر

**تبويب "إحصائيات":**
- إجمالي التذاكر
- تذاكر مفتوحة / قيد المعالجة / مغلقة
- متوسط وقت الحل
- التذاكر حسب النوع (رسم بياني دائري)
- التذاكر حسب الصفحة (أي صفحة فيها أكثر مشاكل)
- متوسط التقييم
- أكثر المستخدمين إرسالاً للتذاكر

### 5.4 إجراءات المسؤولين على التذكرة:

عند فتح تذكرة، المسؤول يرى إضافات:
- **تعيين المسؤول**: قائمة منسدلة لاختيار من يتولى الحل (يمكن تعيين نفسه أو أي مسؤول آخر)
- **تغيير الحالة**: أزرار لتغيير حالة التذكرة
- **تغيير الأولوية**: رفع أو خفض الأولوية
- **إضافة ملاحظة داخلية**: تعليق لا يراه المستخدم العادي (isInternal = true) - يظهر بخلفية صفراء فاتحة للمسؤولين فقط
- **إغلاق التذكرة**: مع كتابة سبب الإغلاق
- **حذف التذكرة**: فقط لمدير الفرع

## القسم 6: الإشعارات

### عند إنشاء تذكرة جديدة:
- إشعار لـ **مدير الفرع** + **رئيس لجنة مراجعة القضايا** + **جميع مستخدمي دور الدعم الفني**: "تذكرة دعم جديدة #[الرقم] من [اسم المستخدم]: [العنوان]"
- إذا الأولوية "عاجل": إشعار urgent

### عند تعيين مسؤول:
- إشعار للمسؤول المعيّن: "تم تعيينك لحل التذكرة #[الرقم]: [العنوان]"
- إشعار لمُرسل التذكرة: "تم تعيين [الاسم] لمتابعة تذكرتك #[الرقم]"

### عند إضافة رد:
- إذا الرد من المسؤول: إشعار لمُرسل التذكرة "رد جديد على تذكرتك #[الرقم]"
- إذا الرد من المُرسل: إشعار للمسؤول "رد جديد من [الاسم] على التذكرة #[الرقم]"

### عند تغيير الحالة:
- إشعار لمُرسل التذكرة: "تم تحديث حالة تذكرتك #[الرقم] إلى: [الحالة الجديدة]"

### عند حل التذكرة:
- إشعار لمُرسل التذكرة: "تم حل تذكرتك #[الرقم] - يرجى تأكيد الحل أو إعادة فتحها"

أضف هذه الأنواع في `NotificationType`:
```typescript
TICKET_CREATED: "ticket_created",
TICKET_ASSIGNED: "ticket_assigned",
TICKET_REPLY: "ticket_reply",
TICKET_STATUS_CHANGED: "ticket_status_changed",
TICKET_RESOLVED: "ticket_resolved",
```

## القسم 10: ويدجت في لوحة التحكم

### للمسؤولين (مدير الفرع + رئيس لجنة المراجعة + دعم فني):
ويدجت "الدعم الفني" يعرض:
- عدد التذاكر الجديدة (لم تُفتح بعد) - بلون أحمر
- عدد التذاكر قيد المعالجة - بلون برتقالي
- عدد التذاكر المعيّنة لي - بلون أزرق
- عدد التذاكر المحلولة هذا الأسبوع - بلون أخضر
- قابل للضغط → ينتقل لصفحة الدعم

### لبقية المستخدمين:
ويدجت صغير يعرض:
- عدد تذاكرك المفتوحة
- آخر رد على تذاكرك (إذا وجد)

## ملخص التدفق:

```
المستخدم يواجه مشكلة أو عنده اقتراح
    ↓
يضغط زر الدعم العائم (أو يدخل صفحة الدعم)
    ↓
يملأ النموذج: النوع + العنوان + الوصف
    ↓
تُنشأ التذكرة (#TK-XXXX) ← إشعار لمدير الفرع
    ↓
المسؤول (مدير الفرع / رئيس لجنة المراجعة / دعم فني) يرى التذكرة → يعيّنها لنفسه أو لغيره ← إشعار
    ↓
محادثة بين المسؤول والمُرسل (ردود متبادلة مع إشعارات)
    ↓
المسؤول يحل المشكلة → يغيّر الحالة لـ "تم الحل" ← إشعار للمُرسل
    ↓
المُرسل يؤكد الحل → التذكرة تُغلق → تقييم (اختياري)
    ↓
[الإحصائيات تُحدّث تلقائياً]
```
